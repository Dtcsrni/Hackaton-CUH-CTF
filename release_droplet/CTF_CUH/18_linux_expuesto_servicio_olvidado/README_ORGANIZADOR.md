# Linux expuesto: servicio olvidado

## Propósito didáctico
Practicar hardening de una unidad `systemd` olvidada.

## Dinámica de resolución
El alumno corrige el archivo de servicio y valida localmente.

## Material del alumno
- Bundle final: `linux_expuesto_servicio_olvidado_bundle.zip`
- Página de apoyo: `/linux-expuesto-servicio-olvidado`

## Validación del organizador
1. Ejecuta `python verify_organizer.py` dentro de esta carpeta.
2. Confirma que el validador devuelve `CUH{servicio_olvidado_documentado_y_limitado}`.
3. Sube el ZIP a CTFd como archivo descargable del reto.
4. Verifica que la página de apoyo publique el contexto correcto.

## Nota pedagógica
Reto de configuración segura sin depender de un host Linux real del alumno.
