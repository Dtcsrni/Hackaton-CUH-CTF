# Portal PHP heredado

## Propósito didáctico
Unir dos prácticas de hardening comunes: prepared statements y validación moderna de contraseñas.

## Dinámica de resolución
El alumno edita un único archivo PHP y valida la corrección de ambos problemas.

## Material del alumno
- Bundle final: `portal_php_heredado_bundle.zip`
- Página de apoyo: `/portal-php-heredado`

## Validación del organizador
1. Ejecuta `python verify_organizer.py` dentro de esta carpeta.
2. Confirma que el validador devuelve `CUH{portal_php_heredado_endurecido}`.
3. Sube el ZIP a CTFd como archivo descargable del reto.
4. Verifica que la página de apoyo publique el contexto correcto.

## Nota pedagógica
El reto sigue siendo de patch local; no requiere base de datos ni runtime PHP para ser evaluado.
