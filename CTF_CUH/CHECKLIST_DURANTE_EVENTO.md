# Checklist durante el evento

- Iniciar o confirmar el arranque de `server_31337.py` antes de permitir intentos del reto de reconocimiento.
- Confirmar que `reto_archivos_linux.zip` sigue disponible para descarga en CTFd.
- Monitorear dudas comunes sobre conectividad, uso de `nmap` y descompresión de archivos.
- Observar tiempos de resolución para ajustar acompañamiento docente sin revelar respuestas.
- Si muchos participantes no encuentran el reto Linux, revisar que el ZIP descargado no esté corrupto y que conserve la estructura esperada.
- Si nadie detecta el puerto, confirmar que la IP publicada en CTFd coincide con la IP real del host y que TCP `31337` sigue en escucha.
- Si el ZIP no descarga, revisar el archivo adjunto en CTFd y volver a cargar `reto_archivos_linux.zip`.
