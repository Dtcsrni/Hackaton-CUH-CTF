Preguntas frecuentes

Manual de operación para resolver el hackatón desde cero

Esta guía está escrita para que puedas resolver el paquete CUH sin depender de explicación oral. Aquí se cruzan plataforma, puertos, adjuntos, nombres de retos y comandos reales: desde Puertas abiertas y Comandos Linux - búsqueda básica hasta Cookie firmada debil, Invitado privilegiado y Formulario de acceso.

Acceso

Cómo entrar y qué recursos forman parte del evento

Antes de abrir un reto conviene ubicar exactamente qué piezas existen: la plataforma HTTPS, los adjuntos publicados y los servicios concretos del host 45.55.49.111.

¿Dónde está la plataforma?

La plataforma oficial del evento está en https://45.55.49.111. Ahí debes iniciar sesión, leer retos, descargar archivos, consultar hints y enviar flags.

La plataforma no requiere VPN. Si no carga, el problema no suele ser de permisos del evento, sino de conectividad local, caché del navegador o bloqueo de tu red.

¿Qué sí forma parte del laboratorio?

  • La propia plataforma CTFd.
  • Los archivos adjuntos publicados dentro de los retos.
  • Los servicios TCP y HTTP expresamente expuestos por el laboratorio.
  • Tu máquina Kali Linux, tu navegador y tus herramientas locales.

¿Qué no forma parte del laboratorio?

  • No tienes acceso SSH al servidor del evento.
  • No tienes acceso a Docker, base de datos, consola, escritorio ni panel del droplet.
  • No debes escanear objetivos fuera de la IP y puertos publicados por el evento.
  • No debes redirigir pruebas a servicios reales o externos.

¿Qué es una flag?

La flag es la respuesta que el sistema acepta como correcta. En este evento todas las flags usan el formato CUH{...}. Aun así, no debes adivinar contenido: primero resuelve el reto y luego envía la respuesta en el formato correcto.

Una flag no se adivina: se obtiene a partir de observación, lectura, enumeración o validación de hipótesis dentro del entorno controlado.

Primeros pasos

Qué debes hacer antes de intentar resolver nada

Checklist inicial

  1. Entrar a https://45.55.49.111 y comprobar que puedes iniciar sesión.
  2. Abrir Reglas y Inicio rápido.
  3. Verificar que tu Kali Linux tiene terminal, navegador y carpeta de descargas accesibles.
  4. Identificar si el reto objetivo usa IP, URL, archivo adjunto, navegador o credenciales débiles.
  5. Elegir la herramienta mínima adecuada antes de lanzar comandos al azar.

Cómo decidir por dónde empezar

Si eres principiante, empieza por Leer también es hacking, Fuente principal, Comandos Linux - búsqueda básica, Puertas abiertas o Cabeceras del laboratorio. Evita saltar de inmediato a Invitado privilegiado, Cookie firmada debil o Formulario de acceso si aún no distingues bien rutas, respuestas y formatos.

Regla de progreso: primero comprende qué superficie estás analizando, luego enumera, después interpreta y al final valida.

Kali Linux

Orientación mínima para no perderte en tu propia máquina

¿Dónde abro la terminal?

Abre la terminal desde el menú de aplicaciones o con el atajo normal de tu entorno gráfico. La terminal será tu centro de trabajo para casi todo el evento.

¿Dónde quedan las descargas?

Normalmente en ~/Descargas o ~/Downloads. Si no sabes cuál existe, prueba:

pwd
ls -la ~
cd ~/Descargas || cd ~/Downloads
pwd
ls -la

Comandos para orientarte

pwd
ls -la
cd carpeta
find . -type f
cat archivo.txt
grep -R "CUH{" .
file nombre_de_archivo
strings archivo.bin | head
Herramientas

Qué herramienta elegir según el tipo de evidencia

nmap

Úsalo cuando el reto te dé una IP o cuando sospeches que un servicio está escuchando en un puerto no estándar.

nmap -sS -Pn 45.55.49.111
nmap -sV -p 31337 45.55.49.111

nc

Te permite conectarte manualmente a un puerto TCP y leer banners, respuestas o líneas de texto devueltas por un servicio.

nc 45.55.49.111 31337

curl

Sirve para revisar páginas web, respuestas JSON, cabeceras, cookies y formularios HTTP.

curl http://45.55.49.111:8082/
curl -i http://45.55.49.111:8088/
curl -X POST http://45.55.49.111:8085/evaluacion -d 'campo=valor'

unzip

Extrae ZIP publicados por los retos. Siempre revisa la estructura interna antes de buscar contenido específico.

unzip reto_archivos_linux.zip
cd reto_linux
find . -type f

find, grep, cat, strings

Son suficientes para muchos retos introductorios. Antes de probar herramientas complejas, confirma si estas ya te dan la respuesta.

hydra

Solo para los retos de fuerza bruta dentro del laboratorio. Antes de usarlo debes identificar si el objetivo usa Basic Auth o formulario.

hydra -l operador -P claves.txt 45.55.49.111 http-get /panel
hydra -l analista -P claves.txt 45.55.49.111 http-post-form "/login:username=^USER^&password=^PASS^:Credenciales invalidas"
Por tipo de reto

Qué hacer según la superficie que estés viendo

Reto con IP o servicio

  1. Verifica que la IP responde.
  2. Enumera puertos con nmap.
  3. Conéctate manualmente al puerto relevante o revisa la respuesta web.
  4. Guarda el hallazgo y compáralo con la descripción del reto.

Reto con ZIP o adjunto

  1. Descarga y localiza el archivo.
  2. Extrae el contenido si es ZIP.
  3. Lista rutas y tipos de archivo.
  4. Busca nombres, textos, patrones y relaciones entre archivos.

Reto web y navegador

  1. Abre la página normal.
  2. Revisa código fuente renderizado, HTML, consola y network.
  3. Observa formularios, cookies, parámetros ocultos y respuestas.
  4. Confirma si el comportamiento depende del cliente o del servidor.

Reto de cracking o fuerza bruta

  1. Identifica el formato: hash, ZIP protegido, Basic Auth o formulario.
  2. Determina usuario, ruta, texto de fallo y diccionario útil.
  3. Prueba con volumen razonable dentro del alcance del reto.
  4. Valida que la respuesta obtenida sea la que pide el enunciado.
DevTools

Cómo usar el navegador como herramienta técnica

Qué revisar en DevTools

  • Elements: HTML renderizado, atributos, comentarios, campos ocultos.
  • Console: mensajes, variables, pistas y errores.
  • Network: rutas, respuestas, códigos HTTP, cabeceras y cookies.
  • Application/Storage: cookies, localStorage y sessionStorage.

Qué preguntas debes hacerte

  • ¿Lo que veo en pantalla es todo lo que el navegador recibió?
  • ¿Hay campos ocultos o cookies influyendo en la respuesta?
  • ¿El servidor valida realmente lo que el cliente envía?
  • ¿Hay pistas en comentarios, consola, JWT, cabeceras o scripts?
Errores comunes

Problemas frecuentes y cómo resolverlos sin detenerte

La flag no valida

Comprueba mayúsculas, llaves, espacios, guiones bajos, comillas y formato exacto. No asumas que toda respuesta sigue el mismo patrón.

No encuentro el archivo descargado

find ~ -maxdepth 3 -type f -name "*.zip"
find ~ -maxdepth 3 -type f | grep reto

501 Unsupported method ('HEAD')

Algunos servicios de laboratorio no implementan HEAD. Usa curl normal con GET o accede desde el navegador.

No sé si debo usar nmap o abrir un archivo

Si el reto te da una IP o una URL, piensa en servicio. Si te da un adjunto, piensa primero en análisis local. Si te da ambas cosas, empieza por entender qué pide realmente la descripción.

Hydra no devuelve resultados

Revisa usuario, ruta, parámetros, método correcto y cadena de fallo. Si el servicio usa Basic Auth, no debes tratarlo como formulario. Si usa formulario, no debes tratarlo como Basic Auth.

El navegador muestra algo distinto a curl

Eso puede significar redirecciones, cookies, JavaScript, cabeceras o comportamiento del cliente. En ese caso revisa Network y compara cada petición.

Cuándo pedir ayuda al organizador

Pide ayuda cuando el bloqueo sea operativo: la plataforma no abre, un archivo no descarga, un servicio no responde, una página pública desapareció o la plataforma rechaza una flag que ya validaste con certeza.

Qué información debes reportar

  • Nombre exacto del reto.
  • Hora aproximada del problema.
  • Comando, URL o acción que ejecutaste.
  • Mensaje exacto de error o captura de pantalla.
  • Si el problema ocurre en Kali, navegador o ambos.
