Fuerza bruta

Hydra con contexto: usuario, ruta, modo y texto de fallo correctos

Esta guía acompaña Acceso por defecto y Formulario de acceso. Ninguno de los dos se resuelve con una ejecución ciega. En ambos debes identificar primero el tipo exacto de autenticación, el usuario, la ruta y el criterio que distingue un intento fallido de uno válido.

Acceso por defecto

El objetivo está en http://45.55.49.111:8090/panel y usa Basic Auth. Si lo tratas como formulario POST, hydra no va a hablar el protocolo correcto.

hydra -l operador -P claves.txt 45.55.49.111 http-get /panel

Formulario de acceso

El objetivo está en http://45.55.49.111:8091/ y sí depende de un POST con parámetros. Aquí debes observar nombres de campos y el texto exacto que marca un fallo.

hydra -l analista -P claves.txt 45.55.49.111 http-post-form "/login:username=^USER^&password=^PASS^:Credenciales invalidas"

Errores que te hacen perder media hora

  • Usar el usuario equivocado.
  • Apuntar a / cuando el formulario real publica en /login.
  • Copiar mal el texto de fallo.
  • Tratar Basic Auth como si fuera HTML o tratar HTML como si fuera Basic Auth.

Qué debes comprobar antes de lanzar Hydra

  1. La ruta exacta que responde.
  2. El usuario esperado por el reto.
  3. El diccionario que el propio reto entrega.
  4. Qué aspecto tiene una respuesta fallida.
