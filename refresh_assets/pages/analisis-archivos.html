Análisis de archivos

Ruta exacta para retos que entregan evidencia descargable

Esta página acompaña retos concretos del paquete CUH: Comandos Linux - búsqueda básica, Logo en observación, Portada con pista, Bitácora del proxy, Hash filtrado y ZIP bajo llave. En todos ellos la pregunta correcta no es “qué herramienta avanzada existe”, sino qué archivo abriste, qué estructura viste y qué texto útil dejaste pasar.

Secuencia mínima que sí funciona

  1. Descarga el adjunto desde CTFd y ubícalo en ~/Descargas o ~/Downloads.
  2. Confirma el tipo real con file si la extensión no basta.
  3. Si es ZIP, extrae el contenido y lista la estructura con find . -type f.
  4. Busca texto útil con grep -R "CUH{" . o con cadenas concretas como admin, proxy, usuario o clave.
  5. Solo después abre archivos específicos con cat o usa strings para imágenes y binarios.

Cómo se comporta cada reto de este bloque

  • Comandos Linux - búsqueda básica: la flag vive en una ruta menos obvia dentro del ZIP.
  • Logo en observación y Portada con pista: el valor útil no está en un texto visible de la UI, sino en el propio archivo de imagen.
  • Bitácora del proxy: la respuesta sale de leer eventos, rutas y contenido textual en un paquete de evidencia.
  • Hash filtrado: el adjunto acota el espacio de búsqueda; no necesitas inventar un ataque fuera del material entregado.
  • ZIP bajo llave: primero identifica la contraseña correcta; la flag está después de abrir el contenedor protegido.
