Cronograma

Secuencia temporal pensada para este paquete de retos

Este cronograma ordena retos que ya existen en la plataforma: primero Leer también es hacking, Fuente principal y Comandos Linux - búsqueda básica; después Puertas abiertas, Cabeceras del laboratorio y JSON de prueba; luego adjuntos, cracking y finalmente autenticación con cookies, JWT y fuerza bruta.

Bloques recomendados

Progresión sugerida de la jornada

Bloque 1. Aterrizaje y lectura del entorno

Entra a la plataforma, revisa reglas, FAQ, inicio rápido y la portada. En este bloque conviene resolver Leer también es hacking, Fuente principal, Consola curiosa y Comandos Linux - búsqueda básica.

Bloque 2. Reconocimiento y validación manual

Pasa a Puertas abiertas, Cabeceras del laboratorio y JSON de prueba, donde debes enumerar puertos, leer banners, revisar cabeceras o validar endpoints HTTP con herramientas manuales.

Bloque 3. Archivos, imágenes y materiales del laboratorio

Trabaja Logo en observación, Portada con pista, Bitácora del proxy, Hash filtrado y ZIP bajo llave. Aquí rinde mucho la combinación de find, grep, cat, file y strings.

Bloque 4. Cliente no confiable y secretos débiles

Deja para esta fase Registro sin servidor, Encuesta confiada, Cookie de rol, Cookie firmada debil, Invitado privilegiado, Secreto compartido debil, Acceso por defecto y Formulario de acceso.

Si vas bien

Cómo escalar dificultad de forma razonable

Señales de que puedes subir nivel

  • Ya entiendes cuándo usar navegador, cuándo usar consola y cuándo usar ambos.
  • Ya resolviste retos con archivos simples o lectura visible.
  • Ya puedes identificar por cuenta propia si un reto usa infraestructura o análisis local.

Siguiente paso lógico

Cuando ya resolviste Leer también es hacking, Comandos Linux - búsqueda básica, Puertas abiertas y Cabeceras del laboratorio, entonces sí conviene pasar a cookies, JWT, cracking o fuerza bruta con Hydra.

Si te atrasas

Cómo recuperar ritmo sin frustrarte

Estrategia de recuperación

  1. Vuelve temporalmente a retos de lectura, archivos simples o HTML visible.
  2. Revisa las páginas de ayuda y confirma si estabas usando la herramienta correcta.
  3. Evita saltar entre demasiados retos al mismo tiempo.
  4. Completa uno, recupera confianza y luego sube de nuevo.

Error frecuente

Perder tiempo en un reto complejo por no haber identificado bien la superficie. Si no sabes si el reto es de red, archivo, navegador o cracking, lo más probable es que todavía no debas forzarlo.

Objetivo pedagógico del cronograma

Que avances desde una pista incrustada en la portada hasta retos donde debes unir varias capas: navegador, terminal, adjuntos, cabeceras, autenticación y validación del backend.

Recomendación final

Resolver `Puertas abiertas`, `Comandos Linux - búsqueda básica`, `Cabeceras del laboratorio` y un reto de frontend con método suele preparar mejor para JWT o cookies que saltar directo a los bloques avanzados por curiosidad.
