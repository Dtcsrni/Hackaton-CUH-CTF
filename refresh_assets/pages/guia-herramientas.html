Guía de herramientas

Qué herramienta usar, cuándo usarla y cómo no perder tiempo

Esta guía está cruzada con retos reales del evento. Si dudas entre abrir Puertas abiertas con nc, revisar Cabeceras del laboratorio con curl -i, buscar una ruta dentro de Comandos Linux - búsqueda básica con find o lanzar hydra sobre Acceso por defecto, esta página te da el criterio exacto.

Antes de correr comandos

Hazte estas preguntas primero

¿Tengo una IP?

Si sí, probablemente estás ante Puertas abiertas, Cabeceras del laboratorio o JSON de prueba. Empieza por nmap, nc o curl, no por Hydra ni por scripts aleatorios.

¿Tengo un archivo?

Si sí, probablemente estás ante Comandos Linux - búsqueda básica, Bitácora del proxy, Hash filtrado o ZIP bajo llave. Empieza por unzip, find, grep, file y strings.

¿Tengo una página web?

Si sí, además de curl necesitas navegador y DevTools para revisar retos como Fuente principal, Consola curiosa, Registro sin servidor, Cookie de rol o Invitado privilegiado.

Reconocimiento y servicios

Herramientas para IP, puertos, banners y respuestas HTTP

nmap

Es la herramienta base cuando un enunciado te deja solo la IP 45.55.49.111 o cuando el puerto útil no aparece en texto. Ese es exactamente el caso de Puertas abiertas.

nmap -sS -Pn 45.55.49.111
nmap -sV -p 31337 45.55.49.111

Cuándo usarlo: cuando el reto habla de observación de red, puertos o conectividad.

nc

Sirve para conectarte manualmente a un servicio TCP y leer lo que responde. Si el reto tiene un banner o un servicio de texto, esta es la forma directa de validarlo.

nc 45.55.49.111 31337

Cuándo usarlo: después de descubrir un puerto interesante.

curl

Te permite revisar respuestas HTTP, cabeceras, redirecciones, cookies, formularios y endpoints JSON en retos como Cabeceras del laboratorio, JSON de prueba, Cookie de rol o Registro sin servidor.

curl http://45.55.49.111:8082/
curl -i http://45.55.49.111:8088/
curl -X POST http://45.55.49.111:8085/evaluacion -d 'campo=valor'

Cuándo usarlo: cuando el reto usa páginas, endpoints o formularios.

Archivos

Herramientas para ZIP, directorios, logs, imágenes y materiales adjuntos

unzip

Extrae archivos ZIP. Nunca asumas lo que hay dentro antes de listarlo.

unzip reto_archivos_linux.zip
cd reto_linux

find

Lista archivos y rutas dentro de una carpeta. Es la forma más rápida de entender estructura.

find . -type f
find . -type d

grep

Busca texto recursivamente. Es útil para ubicar flags, palabras clave, usuarios, rutas o notas internas.

grep -R "CUH{" .
grep -R "admin" .

cat

Muestra el contenido de archivos de texto. Úsalo cuando ya sabes qué archivo quieres leer.

cat README.txt
cat ./evidencia/oculto/flag.txt

file

Identifica el tipo real de un archivo. Es útil cuando una extensión no te dice toda la verdad.

file imagen.png
file evidencia_protegida.zip

strings

Extrae texto legible de binarios, imágenes o formatos no estrictamente de texto. Puede revelar nombres, rutas o flags embebidas.

strings logo_cuh_reto.png | head
strings portada_cuh_reto.png | grep CUH
Navegador y cliente

Herramientas para formularios, cookies, HTML y tokens

DevTools

  • Elements: HTML renderizado, comentarios, atributos y campos ocultos.
  • Console: mensajes y pistas cargadas por scripts.
  • Network: peticiones, respuestas, cabeceras y cookies.
  • Application: cookies, almacenamiento local y valores persistentes.

Cuándo usarlo: siempre que el reto viva en una página o dependa del comportamiento del cliente.

Qué buscar en retos web

  • Comentarios HTML o texto fuera del diseño visible.
  • Mensajes de consola.
  • Campos ocultos en formularios.
  • Cookies con roles, estados o firmas débiles.
  • JWT donde el cliente aparenta tener más control del debido.
Cracking y fuerza bruta

Herramientas que requieren más cuidado operativo

hydra

Úsala solo en Acceso por defecto y Formulario de acceso. Antes de ejecutar nada debes identificar si el objetivo usa Basic Auth o login POST, porque el comando cambia por completo.

hydra -l operador -P claves.txt 45.55.49.111 http-get /panel
hydra -l analista -P claves.txt 45.55.49.111 http-post-form "/login:username=^USER^&password=^PASS^:Credenciales invalidas"

Error común: tratar un Basic Auth como si fuera formulario o viceversa.

Qué revisar antes de lanzar cracking

  1. Formato real del reto: hash, ZIP, Basic Auth, formulario, cookie firmada o JWT.
  2. Usuario esperado o nombre de cuenta.
  3. Cadena de error cuando una prueba falla.
  4. Ruta correcta del endpoint.
  5. Volumen razonable de pruebas dentro del entorno del laboratorio.
Selección rápida

Tabla mental para elegir herramienta

Si el reto muestra una IP

Empieza por nmap. Si detectas un puerto interesante, prueba nc o curl según el protocolo.

Si el reto trae un ZIP

Empieza por unzip, find, grep y cat. Solo después usa algo más específico si la estructura lo justifica.

Si el reto abre una página

Empieza por navegador, DevTools y curl. Ese patrón resuelve retos como Fuente principal, Consola curiosa, Registro sin servidor, Cookie de rol o Invitado privilegiado.

Criterio operativo final

Una herramienta no es “mejor” por ser más compleja. Si find ya te enseña la ruta de Comandos Linux - búsqueda básica, si curl -i ya te muestra un header en Cabeceras del laboratorio o si DevTools ya te revela un valor en Cookie de rol, el siguiente paso correcto es interpretar esa evidencia, no saltar a una técnica más agresiva.

Regla práctica: primero entiende qué superficie tienes enfrente; luego usa la herramienta mínima que responda a esa superficie.
